# Idea: Интеграция с HyperMCPProxy Tool

## Суть идеи
Использование **HSM (Hyper Stack Manager)** в качестве низкоуровневого гипервизора для проекта **HyperMCPProxy**. HSM будет отвечать за динамическое создание изолированных песочниц (Sandboxes) для MCP-серверов.

## Контекст
Проект `mcp-proxy-service` реализует паттерн «Динамического инструментария». Ему необходим надежный механизм для:
1.  **Lazy Loading**: Подгрузки инструментов (MCP-серверов) только в момент вызова.
2.  **Isolation**: Запуска каждого сервера в своей среде (uv venv или node) для предотвращения конфликтов зависимостей.
3.  **Zero-Trust**: Обеспечения безопасности через ограничение прав доступа каждого инструмента.

## Потенциальная польза
- **Масштабируемость**: Возможность управлять тысячами MCP-инструментов без раздувания основного окружения агента.
- **Унификация**: Единый реестр HSM для библиотек, сервисов и теперь — способностей (MCP tools).
- **Изоляция**: Полная физическая изоляция инструментов друг от друга на уровне файловой системы и переменных окружения.

## Техническая реализация (набросок)
- Использование рантаймов `uv` и `node` (планируется) в HSM.
- Вызов `hsm sync` со стороны Proxy Service для подготовки окружения конкретного инструмента.
- Использование флага `--no-workspace` для гарантии автономности каждой песочницы.
