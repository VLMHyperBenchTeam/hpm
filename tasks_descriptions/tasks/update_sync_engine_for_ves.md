# Task: Обновление SyncEngine для поддержки VES

## Контекст
SyncEngine должен научиться материализовать сервисы-репозитории и вызывать правильные адаптеры рантаймов.

## Шаги реализации
1.  **Materialization Logic**:
    - [ ] Реализовать клонирование Git-репозиториев для сервисов в `prod` режиме.
    - [ ] Реализовать вычисление `target_path` для каждого сервиса.
2.  **Orchestration**:
    - [ ] Обновить цикл `sync`: для каждого сервиса определять рантайм и вызывать соответствующий адаптер.
    - [ ] Интегрировать `Implication Merging` для venv-сервисов (проброс параметров через ENV).
3.  **Verification**:
    - [ ] Обновить `EnvironmentInspector` для проверки состояния venv-сервисов.

## Критерии готовности
- [ ] `hsm sync` успешно разворачивает venv-сервис из Git.
- [ ] Переменные окружения корректно пробрасываются в изолированное окружение.
