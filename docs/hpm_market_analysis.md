# Аналитическое сравнение hpm с существующими инструментами

Этот документ содержит внутренние выводы по результатам исследования рынка Python-инструментария на 2026 год.

## 1. Сравнение с лидерами рынка

### uv (Astride / Astral)
*   **Сильные стороны**: Невероятная скорость, замена pip/venv/compile.
*   **Почему hpm нужен поверх uv**: `uv` ориентирован на статическое описание проекта в `pyproject.toml`. Он не умеет динамически подменять источники пакетов (Git vs Local) на основе внешних манифестов без ручного вмешательства в `toml`. `hpm` выступает как "умный оркестратор" для `uv`.

### Hatch
*   **Сильные стороны**: Отличное управление окружениями (environments).
*   **Почему hpm лучше для плагинов**: Hatch требует, чтобы все окружения были описаны в одном файле. В `hpm` каждый плагин автономен. Ты можешь добавить плагин в систему, просто скачав его репозиторий и положив YAML-файл, не трогая основной проект.

### Pixi / Conda
*   **Сильные стороны**: Управление не-Python зависимостями (CUDA, C++).
*   **Ниша hpm**: `hpm` фокусируется на **логике композиции кода**. Мы можем использовать Pixi как бэкенд (вместо `uv`), если проекту нужны сложные системные зависимости, сохраняя при этом гибкость манифестов `hpm`.

## 2. Уникальные преимущества (Unique Selling Points)

1.  **Virtual Packages (Интерфейсы)**: В Python нет стандарта для "интерфейса пакета". `hpm` позволяет ядру зависеть от абстракции `vector-db`, а конкретную реализацию подкладывать в рантайме.
2.  **Zero-Configuration Core**: Репозиторий ядра не "пухнет" от зависимостей плагинов. Это критично для систем, где плагинов могут быть сотни (как в нашем бенчмарке).
3.  **Native Dev/Prod Bridge**: Автоматизация `pip install -e` для локальных папок при наличии `.dev.yaml` — это то, что разработчики обычно делают руками. `hpm` превращает это в стандарт.

## 3. Вывод
`hpm` заполняет пустующую нишу **динамической композиции сред**. Это инструмент не для "простых библиотек", а для **сложных платформ**, где модульность и независимость команд разработки являются приоритетом.
