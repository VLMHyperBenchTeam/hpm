# HSM Installation Guide

Этот документ описывает способы установки **Hyper Stack Manager (hsm)** для различных сценариев.

## 1. Режим разработки (Editable Install)

Используйте этот метод, если вы планируете вносить изменения в сам код HSM. В этом режиме изменения в исходном коде HSM будут мгновенно отражаться на работе команды `hsm` без необходимости переустановки.

### Установка через uv (рекомендуется)
```bash
# Из корня репозитория, где находится папка hsm/
uv pip install -e ./hsm
```

### Проверка
```bash
hsm --help
```

## 2. Режим эксплуатации (Remote Install)

Используйте этот метод для установки стабильной версии HSM в ваш проект напрямую из репозитория GitHub.

### Установка через uv
```bash
uv pip install git+https://github.com/VLMHyperBenchTeam/hsm.git
```

### Установка конкретной версии (тега)
```bash
uv pip install git+https://github.com/VLMHyperBenchTeam/hsm.git@v0.1.0
```

## 3. Использование через uv run (без установки)

Если вы не хотите устанавливать HSM в окружение, вы можете запускать его напрямую:

```bash
uv run --path ./hsm/src/hyper_stack_manager/cli.py --help
```

Или, если вы используете HSM как часть вашего проекта, вы можете вызывать его через `uv run hsm`.
