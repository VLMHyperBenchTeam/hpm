# Задача: Реализация команды hsm package init

## Контекст
Для поддержки паттерна "Self-Bootstrapping Sandbox" и удобства разработчиков, необходима команда для быстрой инициализации новых пакетов.

## Шаги реализации
1.  **Core Logic**:
    *   Добавить метод `init_package(name, path, manager="uv")` в `HSMCore`.
    *   Метод должен:
        *   Создать директорию `path`.
        *   Выполнить `uv init --lib` (или аналогичную команду для выбранного менеджера).
        *   Автоматически зарегистрировать новый пакет в реестре (`hsm-registry/packages/<name>.yaml`) с типом `local` и `editable: true`.
2.  **CLI**:
    *   Добавить команду `init` в группу `package_app` в `hsm/src/hyper_stack_manager/cli.py`.
    *   Аргументы: `name` (обязательный), `--path` (опциональный, по умолчанию `./packages/<name>`).
    *   Флаги: `--register / --no-register` (по умолчанию `--register`). Если флаг установлен, пакет автоматически добавляется в реестр.

## Критерии готовности
*   Команда `hsm package init my-lib` создает валидный Python-пакет.
*   Пакет автоматически появляется в реестре и готов к использованию через `hsm package add`.
