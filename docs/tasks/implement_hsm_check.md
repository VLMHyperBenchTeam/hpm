# Задача: Реализация команды hsm check

## Контекст
Необходимо реализовать команду `hsm check`, которая выполняет полную валидацию текущего проекта и реестра без внесения изменений в файлы или окружение. Это "сухой прогон" (dry-run) перед синхронизацией.

## Шаги реализации
1.  **Core Logic**:
    *   Добавить метод `check()` в `HSMCore`.
    *   Метод должен:
        *   Проверить синтаксис `hsm.yaml`.
        *   Проверить наличие всех указанных пакетов и контейнеров в реестре.
        *   Проверить отсутствие конфликтов в `implies` (например, две опции требуют разные сервисы в одной 1-of-N группе).
        *   Вызвать `docker compose config` для проверки валидности будущего манифеста.
2.  **CLI**:
    *   Добавить команду `check` в основной апп в `hsm/src/hyper_stack_manager/cli.py`.

## Критерии готовности
*   Команда `hsm check` выводит список найденных проблем или сообщение "All checks passed".
*   Команда возвращает код 0 при успехе и отличный от 0 при наличии ошибок (удобно для CI).
