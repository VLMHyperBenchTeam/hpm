# Идея: Обнаружение циклических зависимостей в мета-группах

**Статус**: На будущее
**Дата**: 2026-02-02

## Контекст

В текущей версии (Phase 2) HPM использует плоскую структуру реестра, где группы содержат только пакеты. Зависимости между самими пакетами управляются через `uv`.

## Потенциальный риск

В будущих фазах мы можем внедрить "Мета-группы" (группы, содержащие другие группы). Это создает риск возникновения циклических зависимостей:
*   Группа А включает Группу Б
*   Группа Б включает Группу А

## Предлагаемое решение

Если мета-группы будут реализованы, команда `hpm registry validate` ОБЯЗАНА включать алгоритм поиска циклов (например, поиск в глубину (DFS) в ориентированном графе).

## Пример сценария

```yaml
# registry/groups/meta-vision.yaml
name: meta-vision
includes:
  - group: inference

# registry/groups/inference.yaml
name: inference
includes:
  - group: meta-vision  # ОБНАРУЖЕН ЦИКЛ!
